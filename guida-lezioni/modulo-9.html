Dati reali: strutturati, persistenti e grandi

Fino ad ora abbiamo scritto programmi che lavorano su dati inventati nel codice o inseriti a mano dall'utente. Questo va bene per esercitarsi, ma non è così che funziona il mondo reale.

I dati reali hanno tre caratteristiche che cambiano il modo in cui dobbiamo pensare ai programmi.

Sono persistenti. Nell'architettura di von Neumann, la memoria RAM è volatile: quando il programma finisce, tutto quello che c'era in memoria sparisce. I file esistono sul disco — una memoria secondaria permanente — e sopravvivono alla chiusura del programma. Lavorare con file significa lavorare con dati che esistono prima e dopo l'esecuzione del nostro codice.

Sono potenzialmente grandi. Un file di testo può contenere milioni di righe. Non possiamo aspettarci di caricare tutto in una lista e poi ragionare: dobbiamo scegliere come leggere (tutto in una volta, riga per riga) in base alla dimensione dei dati.

Sono strutturati. I dati raramente arrivano come numeri isolati. Arrivano come tabelle (CSV), dizionari annidati (JSON), testi con marcatura (HTML). La struttura del dato determina anche quale struttura dati Python conviene usare per rappresentarlo.

Recap: file I/O

La funzione open()

Finché i dati sono scritti direttamente nel codice o arrivano da input(), i programmi restano piccoli.

Ma presto vogliamo lavorare su:

  • testi salvati su disco;
  • file di dati;
  • input riutilizzabili.

Per farlo il programma deve poter aprire un file.

nome_file = "data/testo.txt"
file_in = open(nome_file, "r", encoding="utf-8")
print(file_in.read())
file_in.close()

Forma generale:

open(percorso, mode, encoding="utf-8")

Parametri essenziali di open

ParametroSignificato
filepercorso del file
modemodalita', tipicamente r o w
encodingcodifica del testo, spesso utf-8

Punti chiave:

  • il nome del file è una stringa;
  • il percorso può essere assoluto oppure relativo;
  • open(...) restituisce un oggetto che permette di fare operazioni di input/output sul file aperto.

Il costrutto with

Aprire un file non basta: bisogna anche ricordarsi di chiuderlo correttamente.

Se il programma cresce, scrivere sempre:

  • apri;
  • usa;
  • chiudi

può diventare fragile. with serve a rendere questa gestione più sicura e leggibile.

Quando Python esegue un blocco with:

  1. apre la risorsa;
  2. la rende disponibile tramite il nome dopo as (che quindi deve rispettare tutti i constraint su nome di variabile);
  3. esegue il blocco indentato;
  4. alla fine chiude correttamente la risorsa.

read() e readlines()

Una volta aperto un file, dobbiamo decidere come leggerlo.

contenuto = file_handler.read()       # tutto il contenuto come stringa unica
righe = file_handler.readlines()      # lista di righe

for line in file_handler:             # una riga alla volta
    ...

split() e strip()

Quando leggiamo da file otteniamo testo grezzo. Molto spesso ci servono unità più piccole: righe, campi, parole.

Ogni riga di un file termina con un carattere speciale chiamato newline, scritto \n. Non è una barra e una n: è un singolo carattere invisibile che rappresenta "vai a capo". Quando Python legge una riga da file, questo carattere è incluso nella stringa.

riga = "Sandra,Rossi,informatica,30\n"   # il \n è il fine riga

riga.split(",")          # ['Sandra', 'Rossi', 'informatica', '30\n']  — il \n rimane nell'ultimo campo
riga.strip()             # 'Sandra,Rossi,informatica,30'               — strip() rimuove il \n
riga.strip().split(",")  # ['Sandra', 'Rossi', 'informatica', '30']    — risultato pulito

Schema tipico:

somma = 0

with open("dati.txt", "r", encoding="utf-8") as file_in:
    for line in file_in:
        somma = somma + int(line.strip())

strip() senza argomenti toglie i caratteri bianchi ai bordi. Se serve, possiamo anche specificare quali caratteri togliere:

line.strip(".,;")

Scrivere su un file

Forma base:

with open("output.txt", "w", encoding="utf-8") as file_output:
    print("ciao", file=file_output)

Attenzione:

  • w sovrascrive il contenuto precedente;
  • il nome del file è sempre una stringa che rappresenta un percorso.

print(...) di default scrive sullo standard output. Con file=... possiamo mandare lo stesso output in un file aperto.

Alternativa: redirezione dalla shell:

python3 script.py > output.txt

Nel secondo caso è la shell che intercetta lo standard output. Nel primo caso è il programma Python che decide dove scrivere.

sys.argv

Se il nome del file di input e di output è scritto nel codice, ogni volta che cambia dobbiamo modificare lo script. Molto meglio passare questi dati dall'esterno, da riga di comando.

import sys

nome_input = sys.argv[1]
nome_output = sys.argv[2]

sys.argv è una lista che contiene gli argomenti passati al programma da riga di comando:

  • sys.argv[0] è il nome dello script;
  • sys.argv[1] e i successivi sono gli argomenti forniti dall'utente.

Standard input, open() e sys.argv

Ci sono tre modi diversi per far arrivare dati a un programma.

1. Standard input

numero = int(input("Inserisci un numero: "))

2. File aperto con open()

with open("input.txt", "r", encoding="utf-8") as file_in:
    for line in file_in:
        numero = int(line.strip())

3. Parametri da riga di comando

import sys

numero = int(sys.argv[1])

Esplorazione rapida da terminale

Quando lavoriamo con file di dati, spesso abbiamo bisogno di capire cosa c'è dentro prima di scrivere uno script. Il problema è che non tutti i file si possono aprire con un programma con interfaccia grafica.

Un file CSV o di testo si può aprire con Excel o con un editor di testo, ma un file con milioni di righe potrebbe bloccare il programma, o il file potrebbe avere un formato che il programma non riconosce. In molti contesti reali — server remoti, ambienti di calcolo, pipeline di dati — non c'è nemmeno un'interfaccia grafica disponibile.

Il terminale permette di ispezionare qualsiasi file di testo in modo rapido e controllato, indipendentemente dalla sua dimensione o dal formato.

bash fornisce delle utility utili per:

  • ispezionare rapidamente un file;
  • fare piccole trasformazioni;
  • capire il procedimento logico da seguire prima di scrivere uno script più completo.
ComandoUso tipico
cat file.txtmostra tutto il file
less file.txtscorre il file una schermata alla volta
head file.txtmostra l'inizio del file
tail file.txtmostra la fine del file
grep pattern file.txtfiltra le righe che contengono un pattern
cut ...seleziona campi o colonne

La pipe |

Ogni utility da terminale fa una cosa sola: wc conta, sort ordina, grep filtra. La potenza vera arriva quando si collegano più comandi in sequenza.

La pipe | collega lo standard output di un comando allo standard input del successivo, senza creare file intermedi:

comando1 | comando2 | comando3

L'output di comando1 diventa l'input di comando2, l'output di comando2 diventa l'input di comando3, e così via. È come una catena di montaggio: ogni passaggio riceve il risultato del precedente e lo trasforma.

Esempio: quante parole distinte contiene un testo?

cat testo.txt | tr ' ' '\n' | sort | uniq | wc -l

Passo per passo:

  • cat testo.txt — stampa il file intero
  • tr ' ' '\n' — sostituisce ogni spazio con un a capo: una parola per riga
  • sort — ordina le righe alfabeticamente (necessario perché uniq lavora su righe consecutive)
  • uniq — elimina le righe duplicate consecutive
  • wc -l — conta le righe rimaste

Se invece vogliamo salvare il risultato in un file si usa >:

cat testo.txt | tr ' ' '\n' | sort | uniq > parole-uniche.txt

Formati di input/output

Testo semplice — una riga dopo l'altra; nessuna struttura esplicita oltre ai caratteri e ai separatori.

CSV — valori separati da virgole o altro separatore; ogni riga rappresenta spesso un record.

JSON — struttura annidata; tipicamente oggetti, liste, chiavi e valori.

HTML — testo con marcatura; utile quando vogliamo estrarre informazioni da pagine o frammenti strutturati.

Di solito esistono libreria apposite per leggere ognuno di questi formati ma a noi importa:

  • capire che tipo di struttura ha il dato;
  • scoprire quale metodo di lettura e segmentazione conviene usare;
  • che struttura dati usare: quando basta il testo semplice e quando serve una rappresentazione più ricca.

Dati strutturati

Immaginiamo di avere un file con tutti i voti di tutti gli studenti per tutte le materie:

Sandra,Rossi,matematica,28
Matteo,Bianchi,matematica,22
Sandra,Rossi,informatica,30
Sandra,Rossi,linguistica,27
Matteo,Bianchi,informatica,19
Matteo,Bianchi,linguistica,25
...

Problema 1: Vogliamo calcolare la media dei voti per ogni studente. Il problema è che ogni studente compare su più righe: dobbiamo raccogliere tutti i suoi voti mentre leggiamo il file, e poi calcolare la media alla fine.

Con una lista non possiamo farlo facilmente: la lista non ha un modo diretto per rispondere alla domanda "dammi tutti i voti di Sandra Rossi". Dovremmo scorrere tutta la lista ogni volta.

Ci serve una struttura che associ un nome (la chiave) a una lista di voti (il valore). Questa struttura è il dizionario.

voti = {
    "Sandra Rossi":  [28, 30, 27],
    "Matteo Bianchi": [22, 19, 25],
}

Problema 2: Vogliamo la lista di tutti gli studenti distinti.

Ogni studente compare su più righe. Se raccogliamo i nomi in una lista man mano che leggiamo il file, otterremo duplicati:

["Sandra Rossi", "Matteo Bianchi", "Sandra Rossi", "Sandra Rossi", "Matteo Bianchi", ...]

Potremmo controllare ogni volta se il nome è già nella lista prima di aggiungerlo, ma su file grandi diventa lento. Ci serve una struttura che tenga solo elementi distinti: il set.

studenti = set()

studenti.add("Sandra Rossi")
studenti.add("Matteo Bianchi")
studenti.add("Sandra Rossi")   # già presente: non viene aggiunto

# studenti → {"Sandra Rossi", "Matteo Bianchi"}

Dizionari

Un dizionario si scrive tra { e }. Ogni elemento è una coppia chiave: valore separata da virgola:

{}                                       # dizionario vuoto
{"the": 156, "cat": 20, "is": 80}       # chiavi stringa, valori interi
{"lazio": "roma", "campania": "napoli"} # chiavi e valori stringa
{"molise": ["campobasso", "isernia"]}   # valore che è una lista

Per creare un dizionario vuoto e popolarlo un elemento alla volta si usa l'assegnazione con []:

d = {}                 # dizionario vuoto
d["chiave"] = valore   # aggiunge (o aggiorna) la coppia chiave → valore

La chiave tra [] deve essere un valore immutabile (stringa, intero, …). Il valore a destra può essere qualunque espressione Python. Se la chiave esiste già, il valore viene sovrascritto:

d = {}
d["gatto"] = 1
d["gatto"] = 5   # ora d["gatto"] vale 5, non 1

Semantica

Un dizionario associa una chiave a un valore:

  • la chiave deve essere immutabile (stringa, intero, tupla) — non si possono usare liste o altri dizionari come chiavi;
  • il valore può essere di qualunque tipo, incluse liste o altri dizionari;
  • l'accesso avviene tramite la chiave, non tramite la posizione.

Se si accede a una chiave che non esiste si ottiene un KeyError:

d = {"gatto": 1}
d["cane"]   # KeyError: 'cane'

Per evitarlo si usa prima in:

if "cane" in d:
    print(d["cane"])

Lista o dizionario?

Una lista può contenere le stesse informazioni, ma il significato dei campi resta implicito nelle posizioni:

persona_lista = ["Ludovica", "Pannitto", 32]

Con un dizionario la struttura diventa più leggibile e meno fragile:

persona = {"nome": "Ludovica", "cognome": "Pannitto", "eta": 32}

Iterazione sui dizionari

Iterare sulle chiavi (default):

for chiave in dizionario:
    print(chiave, dizionario[chiave])

Iterare su coppie chiave-valore con items():

for chiave, valore in dizionario.items():
    print(chiave, valore)

items() restituisce coppie (chiave, valore) che Python spacchetta direttamente nelle due variabili del for.

Si può iterare un dizionario con il while? No. Il while richiede una condizione booleana e un aggiornamento manuale — non sa come scorrere una struttura dati.

Problema completo: media dei voti per studente

Il file dati/voti_materie.txt contiene i voti di tutti gli studenti per tutte le materie. Vogliamo calcolare la media dei voti per ogni studente e l'elenco delle materie presenti.

Passo 1: leggere il file riga per riga

Ogni riga ha quattro campi separati da virgola:

with open("dati/voti_materie.txt", "r", encoding="utf-8") as f:
    for line in f:
        nome, cognome, materia, voto = line.strip().split(",")
        studente = nome + " " + cognome
        voto = int(voto)

Passo 2: raccogliere i voti per studente

Ogni studente compare su più righe. Usiamo un dizionario studente → lista di voti:

voti = {}

with open("dati/voti_materie.txt", "r", encoding="utf-8") as f:
    for line in f:
        nome, cognome, materia, voto = line.strip().split(",")
        studente = nome + " " + cognome
        voto = int(voto)

        if studente not in voti:
            voti[studente] = []
        voti[studente].append(voto)

Dopo la lettura del file, voti contiene:

{
    "Sandra Rossi":   [28, 30, 27],
    "Matteo Bianchi": [22, 19, 25],
    ...
}

Passo 3: calcolare la media per ogni studente

for studente, lista_voti in voti.items():
    media = sum(lista_voti) / len(lista_voti)
    print(studente, round(media, 2))

Passo 4: raccogliere le materie distinte

Se usassimo una lista, ogni materia comparirebbe più volte. Usiamo un set:

materie = set()

with open("dati/voti_materie.txt", "r", encoding="utf-8") as f:
    for line in f:
        nome, cognome, materia, voto = line.strip().split(",")
        materie.add(materia)

print(materie)  # {'fisica', 'informatica', 'linguistica', 'matematica', 'statistica'}

Set

Un set è una collezione di elementi unici e non ordinati:

  • non mantengono l'ordine;
  • non ammettono duplicati;
  • sono modificabili;
  • gli elementi al loro interno devono essere immutabili.
s = {1, 2, 3, 2, 1}   # {1, 2, 3} — i duplicati vengono rimossi
s = set()              # set vuoto (non {} che è un dizionario vuoto)

Esempi:

{3, 4, 5, 6}
{"elefante", "cammello", "pentola"}

set("linguistica")
set(["elefante", "cammello", "pentola", "elefante"])

set(lista) è spesso il modo più rapido per eliminare i duplicati da una sequenza.

Quando usare un set

Un set è utile quando:

  • si vuole sapere se un elemento è già stato visto (senza contarlo);
  • si vuole eliminare i duplicati da una sequenza;
  • si vogliono confrontare due insiemi (chi è in entrambi, chi manca).
# eliminare duplicati da una lista
parole = ["gatto", "cane", "gatto", "topo", "cane"]
uniche = set(parole)   # {"gatto", "cane", "topo"}

Scegliere tra lista, dizionario e set

StrutturaQuando usarla
listconserva ordine e ripetizioni, accedi per posizione
dictassocia chiavi a valori, cerchi per nome non per posizione
setinsiemi, controllo di appartenenza, eliminazione duplicati

Esempi pratici:

  • vocabolario senza ripetizioni → set
  • sequenza da mantenere in ordine → list
  • rubrica parola → frequenza → dict
  • confronto tra parole di due testi → set
